# IntentLink 선언문 (The IntentLink Manifesto) v1.0.0
> "정적 링크에서 살아있는 연결로: 자율 웹(Autonomous Web)의 여명"

## 1. 문제 제기: 파편화된 웹과 AI의 한계
현재의 웹은 인간을 위한 **하이퍼링크(Hyperlink)**로 연결되어 있습니다. AI 에이전트 시대에 이는 두 가지 큰 장벽을 만듭니다.

### Q. "LLM이 다 알아서 해주지 않나요?"
많은 사람들이 묻습니다. "ChatGPT한테 짜달라고 하면 되는데 굳이?"
하지만 LLM과 RAG(검색 증강 생성)에는 명확한 한계가 있습니다.

1.  **환각(Hallucination):** LLM은 과거의 데이터로 확률적인 코드를 짭니다. API 버전이 바뀌면 오작동합니다.
2.  **비정형의 불확실성:** RAG는 HTML 문서를 읽고 '추측'합니다. "이게 POST인지 GET인지", "필수 파라미터가 뭔지" 모호한 경우가 많습니다.
3.  **실행의 위험성:** AI는 "이 API를 호출하면 100달러가 결제된다"는 사실을 인지하지 못한 채 코드를 실행할 수 있습니다.

## 2. 해결책: IntentLink 프로토콜
**IntentLink**는 AI를 위한 **'실시간 내비게이션 지도'**입니다. 모호한 자연어가 아닌, 기계가 읽을 수 있는 **약속된 언어(Protocol)**로 소통합니다.

### 핵심 아키텍처
1.  **매니페스트 (`link.json`):** 서비스의 DNA입니다. "나는 희귀 도서를 팝니다(Purpose)", "재고 검색이 가능합니다(Offers)", "배송이 필요합니다(Needs)"를 명확한 JSON으로 정의합니다.
2.  **브로커 (The Brain):** 이 명세서를 읽고 "서점에는 배송사가 필요하군요"라고 판단하여 연결하는 지능형 중개자입니다.
3.  **러너 (The Hands):** 인간의 승인(Human-in-the-loop) 하에 안전하게 코드를 실행하는 엔진입니다.

## 3. 왜 IntentLink인가? (RAG vs IntentLink)
IntentLink는 RAG와 경쟁하는 것이 아니라, **RAG를 위한 최적의 정형 데이터(Structured Context)**를 제공합니다.

| 구분 | 일반 RAG (HTML 문서 검색) | IntentLink (link.json) |
| :--- | :--- | :--- |
| **데이터** | 비정형 (HTML, 텍스트) | **정형 (JSON 스키마)** |
| **정확도** | 추론 및 확률적 접근 | **확정적 실행 규칙** |
| **안전성** | 위험성 판단 어려움 | **Side-effect 명시로 안전 확보** |
| **속도** | 문서 파싱에 많은 토큰 소모 | **최소 토큰으로 즉시 파악** |

## 4. 생태계: 모두가 연결되는 세상
IntentLink는 단순한 기술 규격이 아닌, 살아있는 생태계입니다.

- **서비스 제공자:** `/.well-known/link.json` 파일 하나만 올리면 전 세계 모든 AI가 당신의 서비스를 즉시 사용할 수 있습니다.
- **AI 에이전트:** 복잡한 학습 없이도, 처음 보는 서비스를 즉시 이해하고 활용할 수 있습니다.
- **개발자:** 지루한 연동 코드 작성(Glue Code)에서 해방됩니다.

## 5. 미래 비전: 자율 진화 (Autonomous Evolution)
레지스트리가 커질수록, 웹은 스스로 치유하고 최적화하는 **'자율 신경망'**이 됩니다.
배송 API가 다운되면, 브로커는 즉시 대체 가능한 다른 배송사를 찾아 연결할 것입니다. 인간의 개입 없이도 시스템은 계속 돌아갑니다.
